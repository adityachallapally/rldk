# Contract-Based API Refactoring Implementation

This document summarizes the implementation of a contract-based approach to guide API refactoring without relying on documentation.

## Overview

The implementation provides a systematic way to:
1. Discover the current API surface through code introspection
2. Define a target API surface (v1 vision)
3. Track progress toward the target through automated diffs
4. Ensure no documentation is read during the process

## Files Created

### Core Contract Files
- `api_contract.baseline.yaml` - Auto-generated baseline contract from current code
- `api_contract.v1.yaml` - Hand-written v1 vision contract
- `artifacts/api_contract.diff.txt` - Automated diff between baseline and v1

### Schema Files
- `schemas/determinism.schema.json` - JSON schema for determinism artifacts
- `schemas/reward_health.schema.json` - JSON schema for reward health artifacts  
- `schemas/repro.schema.json` - JSON schema for repro artifacts

### Test Files
- `tests/test_contract_baseline.py` - Discovers and validates baseline contract
- `tests/test_contract_v1_diff.py` - Computes diff between baseline and v1
- `tests/test_no_docs_in_contract.py` - Guards against reading documentation
- `tests/test_golden_master_contract_validation.py` - Validates golden master against contracts

### CI Configuration
- `.github/workflows/contract-tests.yml` - GitHub Actions workflow for contract validation

## How It Works

### 1. Baseline Contract Discovery (`test_contract_baseline.py`)

The baseline contract is generated by:
- **Symbol Discovery**: Imports `rldk` package and inspects `__all__` to find public symbols
- **CLI Discovery**: Runs `rldk --help` and parses command names, then runs `rldk <cmd> --help` for each command
- **Artifact Discovery**: Runs minimal synthetic tests to discover output formats and artifact schemas

### 2. V1 Vision Contract (`api_contract.v1.yaml`)

The v1 contract defines the target API surface with:
- **Symbols**: Core classes and functions (ExperimentTracker, TrackingConfig, scan_ppo_events)
- **CLI**: Simplified command interface (determinism, reward_health, repro)
- **Artifacts**: JSON schemas for required fields

### 3. Diff Tracking (`test_contract_v1_diff.py`)

The diff test computes human-readable differences:
- **Symbols**: Added/removed/changed symbols with kind and docstring info
- **CLI**: Added/removed/changed commands with synopsis and flags
- **Artifacts**: Added/removed/changed artifact patterns and schemas

### 4. Documentation Guard (`test_no_docs_in_contract.py`)

The guard test ensures no documentation is read by:
- Monitoring file opens during contract tests
- Detecting documentation file patterns (.md, .rst, README, docs/, etc.)
- Failing tests if documentation is accessed

### 5. Golden Master Validation (`test_golden_master_contract_validation.py`)

The golden master test validates that:
- Commands produce expected exit codes
- Output matches recorded regex patterns
- Artifacts validate against JSON schemas
- All contract requirements are met

## Usage

### Generate Baseline Contract
```bash
python3 tests/test_contract_baseline.py
```

### Compute Diff
```bash
python3 tests/test_contract_v1_diff.py
```

### Run All Tests
```bash
python3 -m pytest tests/test_contract_*.py -v
```

### CI Integration
The GitHub Actions workflow automatically:
1. Runs baseline contract tests
2. Computes and publishes diff
3. Validates no documentation is read
4. Comments PRs with diff information

## Benefits

1. **No Documentation Dependency**: Tests learn API surface from live code only
2. **Clear Progress Tracking**: Diff shows exactly how far you are from target
3. **Automated Validation**: CI ensures contracts stay in sync with code
4. **Refactoring Safety**: Golden master tests prevent breaking changes
5. **Explicit Vision**: V1 contract clearly defines the target API surface
